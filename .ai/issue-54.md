## Требования

### 1. Новая функция группировки
В модуле `src/reporting/summary.py` добавить функцию:

`group_notes_by_tag(notes: list[Note]) -> dict[str, list[Note]]`

Поведение:
- вернуть словарь, где ключ — тег, значение — список заметок с этим тегом
- если у заметки несколько тегов, она должна попасть в каждую соответствующую группу
- заметки без тегов в результат не включать
- порядок заметок внутри каждой группы должен сохранять исходный порядок входного списка

### 2. Расширение reporting CLI
В `src/reporting/cli.py` добавить флаг:

`--show-grouped`

Если флаг передан:
- после текущего summary выводить группы заметок по тегам
- формат вывода строго такой:

```text
total=3
draft=1
home=1
work=2

[group:draft]
[3] C :: work,draft :: z

[group:home]
[2] B :: home :: y

[group:work]
[1] A :: work :: x
[3] C :: work,draft :: z
```

Требования к выводу:
- сначала остаётся текущий вывод `total=...` и статистика по тегам
- затем пустая строка
- затем группы в порядке сортировки тегов по возрастанию
- внутри группы заметки выводятся в том же формате, что и в notebook CLI:
  `[id] title :: tag1,tag2 :: text`

### 3. Повторное использование форматирования
Не дублировать форматирование заметки в `src/reporting/cli.py`, если можно переиспользовать уже существующую функцию форматирования из notebook-модуля без архитектурно плохих зависимостей.
